配置 Anti-Xray

最初由 stonar96 编写和维护。

Paper 包含一个基于混淆的 Anti-Xray 系统，具有三种模式，可以按世界配置。

[按世界配置]

如果你还不熟悉按世界配置，请花点时间熟悉一下配置指南。

本指南是配置 Anti-Xray 的逐步指南。有关参考文档，请参阅 按世界配置参考中的 Anti-Xray 部分。

Anti-Xray 有三种不同的模式。engine-mode: 1 将指定的方块（hidden-blocks）替换为其他“假”方块，如stone（在 y < 0 时为deepslate）、netherrack或end_stone，具体取决于维度。相比之下，engine-mode: 2 将同时替换hidden-blocks和replacement-blocks为随机生成的hidden-blocks。engine-mode: 3 的工作方式类似于engine-mode: 2，但不是随机化每个方块，而是随机化每个区块的每一层。

以下图片1显示了在推荐配置下，使用 Xray 的玩家在主世界和下界中看到的每种模式的效果。

特别是在客户端方面，engine-mode: 1 的计算量要小得多，而engine-mode: 2 可能更能防止 Xray。使用engine-mode: 1，只有完全被实心方块覆盖的矿石才会被隐藏。在洞穴中的空气或湖泊中的水暴露的矿石不会被隐藏。使用engine-mode: 2，假矿石会阻挡真实方块的视线。如果将air添加到hidden-blocks中，engine-mode: 2将有效地隐藏所有矿石，即使是暴露在空气中的矿石。engine-mode: 3可以在加入时减少约 2 倍的网络负载，并有助于区块包的压缩。

[Anti-Xray 绕过]

范围扩展：虽然 Anti-Xray 本身可以防止大多数用户在你的服务器上使用 Xray，但它并不是万无一失的。由于 Anti-Xray 的实现方式（以及必须的实现方式），在默认服务器上，可以将你能看到的真实矿石的范围扩展到一个不小的量。这可以通过任何有能力的反作弊插件来缓解；然而，这并不是开箱即用的。

种子反转：另一个攻击向量是 Minecraft 世界生成的确定性。如果客户端能够获得世界种子，它就能知道每个生成的矿石的真实位置，完全绕过 Anti-Xray。这可以通过使用feature-seeds配置以及spigot.yml中的结构种子选项来部分解决。请注意，这不是一个完整的解决方案，客户端仍然可能获得服务器的世界种子。为每个世界使用不同的种子也可能是有益的。

暴露在空气中的矿石：在engine-mode: 1、engine-mode: 2和engine-mode: 3中，客户端可能会看到暴露在空气中的矿石。这可以通过在engine-mode: 2和engine-mode: 3中将air添加到hidden-blocks列表中来缓解。然而，这样做可能会导致某些玩家的客户端性能问题（FPS 下降）。

engine-mode: 1​

默认世界配置

将paper-world-defaults.yml中的现有anticheat.anti-xray块替换为以下内容：

paper-world-defaults.yml
anticheat:
  anti-xray:
    enabled: true
    engine-mode: 1
    hidden-blocks:
      # 地牢箱子完全被空气包围，因此无法隐藏，但埋藏的宝藏将被隐藏。
      - chest
      - coal_ore
      - deepslate_coal_ore
      - copper_ore
      - deepslate_copper_ore
      - raw_copper_block
      - diamond_ore
      - deepslate_diamond_ore
      - emerald_ore
      - deepslate_emerald_ore
      - gold_ore
      - deepslate_gold_ore
      - iron_ore
      - deepslate_iron_ore
      - raw_iron_block
      - lapis_ore
      - deepslate_lapis_ore
      - redstone_ore
      - deepslate_redstone_ore
    lava-obscures: false
    # 从1.18开始，一些矿石生成得更高。
    # 请根据自己的判断调整max-block-height设置。
    # https://minecraft.wiki/w/Ore 可能会有所帮助。
    max-block-height: 64
    # replacement-blocks列表在engine-mode: 1中不使用。更改此项将无效。
    replacement-blocks: []
    update-radius: 2
    use-permission: false
下界配置

复制并粘贴到下界世界文件夹中的paper-world.yml中。有关更多信息，请参阅配置指南。

world_nether/paper-world.yml
anticheat:
  anti-xray:
    enabled: true
    engine-mode: 1
    hidden-blocks:
      - ancient_debris
      - nether_gold_ore
      - nether_quartz_ore
    lava-obscures: false
    max-block-height: 128
    # replacement-blocks列表在engine-mode: 1中不使用。更改此项将无效。
    replacement-blocks: []
    update-radius: 2
    use-permission: false

常见问题、常见陷阱和支持​

我仍然能看到（一些）矿石/使用X-ray

如上所述，即使你启用了 Anti-Xray，仍然可能看到（一些）矿石的几个原因：

  • 矿石位于配置的max-block-height值之上。
  • Anti-Xray 无法隐藏暴露在空气或其他透明方块（例如洞穴中的空气）中的矿石。原则上，这对于engine-mode: 2和engine-mode: 3也是如此，然而，通常假矿石会阻挡真实方块的视线。隐藏这些暴露的矿石需要额外的插件。
  • use-permission选项已启用，并且你拥有 Anti-Xray 绕过权限（paper.antixray.bypass）或你是操作员。
  • 配置的方块列表中缺少方块类型。这可能是由于使用了过时的配置文件。
我添加了假方块，但X-ray没有显示它们

如果你使用engine-mode: 2或engine-mode: 3并且你已将假方块添加到hidden-blocks列表中，但在游戏中使用 X-ray 看不到它们，这可能有以下原因：

  • 添加的方块类型是 tile 实体。Anti-Xray 可以隐藏（替换）tile 实体（如箱子），前提是它们没有暴露在空气或其他透明方块中。然而，Anti-Xray 不能将 tile 实体作为假方块放入区块中。
  • 你的客户端的 X-ray mod 禁用了该方块或你的 X-ray 资源包没有显示该方块。
它在y = 0以下或某些其他地方不起作用。
  • 你的配置文件可能已过时，缺少replacement-blocks列表中的重要方块，如deepslate或特定生物群系的方块，如basalt。你可能还需要检查hidden-blocks列表是否包含所有重要的矿石及其deepslate变体。
  • 如果它在某个 y 级别以上不起作用，请检查你的max-block-height设置。
它仍然不起作用，进一步排除故障
  • 确保在更改 Anti-Xray 配置后始终重新启动服务器。更改不会自动应用。
  • 不要使用/reload命令。要应用 Anti-Xray 配置更改，需要重新启动。
  • 重新启动服务器后，通过使用 timings 或 spark 检查配置部分，验证配置是否正确应用。
如果它仍然不起作用，我该如何以及在哪里寻求支持？

如果上述要点没有解决你的问题，或者你对 Anti-Xray 有进一步的问题，请不要犹豫，在PaperMC Discord的#paper-help 频道向我们提问。请尽量提供尽可能多的关于你的问题的细节。“它不起作用”在寻求支持时并不是很有帮助。描述你想要实现的目标，你尝试了什么，你期望什么以及你观察到了什么。最好包括一个 timings 或 spark 链接以及你在游戏中观察到的图片。
